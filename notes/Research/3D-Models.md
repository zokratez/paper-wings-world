# 3D Paper Plane Models — Current State & Upgrade Path

## Current State (Phase 2)

We currently have **8 low-poly rigged paper plane prefabs** generated by the editor tool:

- `Assets/Prefabs/Planes/`

### Models Generated
- Classic Dart
- The Ring
- Loop Plane
- Nakamichi Glider
- Stealth Glider
- Light Spinner
- Canard
- The Bird

### Quality Level
- Low-poly but intentionally clean and readable.
- Proper bone hierarchy matching the `PaperPlaneAnimator` expectations:
  - `Body`
  - `LeftWing` / `RightWing`
  - `LeftWingTip` / `RightWingTip`
  - `Tail`
- Improved geometry with subtle crease geometry on wings for a more "real folded paper" appearance.
- Each plane has distinct proportions and shaping so they feel different even at low poly.

### Assignment
All 8 `PaperPlaneDefinition` assets now reference their corresponding real 3D model prefabs by default (via the "Assign Real Models to All PaperPlaneDefinitions" tool).

## How the System Works

- `PaperPlaneDefinition.paperModelPrefab` holds the reference.
- When the `FoldingTutorialManager` loads a plane, it checks for a real prefab first.
- If present, it instantiates the prefab + adds `PaperPlaneAnimator`.
- If not present (fallback), it uses the old `ProceduralPaperPlane`.

This design allows seamless swapping between placeholder and final art.

## Future Upgrade Path (Recommended)

When we are ready for higher visual quality:

1. **Keep the exact bone hierarchy** (do not rename `Body`, `LeftWing`, etc.).
2. In Blender (recommended):
   - Import the current low-poly prefab.
   - Replace the primitive meshes with proper low-poly paper geometry.
   - Add subtle bevels, better edge loops for creases, and slight thickness.
   - Keep the same bone weights or re-bind cleanly.
3. Export as `.fbx` (with "Bake Animation" off if using the runtime animator).
4. Replace the mesh in the prefab or create a new higher-quality variant.
5. Update the `PaperPlaneDefinition` if you want to keep both versions (e.g., "Low" vs "High" quality).

### Recommended Naming Convention for Future Variants
- `classic_dart_low.prefab`
- `classic_dart_high.prefab`

The `PaperPlaneDefinition` can then be updated to point at the desired quality level.

## Notes for Artists

- Use **Unlit** or simple **Lit** materials with a paper texture.
- Avoid heavy normal maps — paper should read as flat with good silhouette and edge definition.
- Keep polygon count reasonable (aim for 300–800 tris per plane for mobile).
- Test folding animations frequently while modeling.
